# Contributing to NileGazer

Thanks for helping improve NileGazer.

## Standards

Contributions should be:
- focused,
- well-tested,
- documented when relevant,
- and consistent with the project architecture.

## Ways to contribute

- Fix bugs.
- Improve documentation.
- Add tests.
- Build adapters.
- Improve examples.
- Strengthen safety and policy checks.
- Suggest roadmap items.

## Before you start

- Read the README and roadmap.
- Check open issues.
- Look for `good first issue` and `help wanted`.
- Keep pull requests small and reviewable.

## Workflow

1. Create a branch.
2. Make the change.
3. Run the build.
4. Open a pull request.

## Commit style

Use short, descriptive commit messages.
